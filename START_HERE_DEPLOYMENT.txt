Copy-Paste vs Typing:
  ✓ COPY AND PASTE the commands
  ✗ Don't type them manually
  ✗ Typing leads to mistakes

Folder Organization:
  ✓ Create the exact folder structure shown
  ✗ Don't just dump files in one folder
  ✗ Wrong structure = deployment fails

Git Commands:
  ✓ Run commands ONE AT A TIME
  ✓ Wait for each to finish before next
  ✗ Don't run multiple commands together

Vercel Deployment:
  ✓ Wait for "Success!" message
  ✗ Don't assume it's done too quickly
  ✗ Takes 1-2 minutes usually

🆘 STUCK? TRY THIS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: Command Prompt won't open
  → Right-click Command Prompt
  → Run as Administrator

Problem: "git is not recognized"
  → You need to install Git first
  → See: GIT_INSTALL_WINDOWS.md

Problem: "cd: command not found" (on Mac)
  → Use: cd ~/Documents/cap-springfield-inventory
  → Notice the ~ instead of path

Problem: Can't find your folder
  → Try: cd Desktop\cap-springfield-inventory
  → Or: cd Downloads\cap-springfield-inventory
  → Check where you saved it

Problem: GitHub push asks for password
  → Use your GitHub password (not your email)
  → Or use a personal access token if 2FA enabled
  → See GitHub settings for token setup

Problem: Vercel deployment fails
  → Check GitHub is up to date (git push worked)
  → Check folder structure is correct
  → Look at Vercel build logs for errors
